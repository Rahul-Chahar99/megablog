Project Structure Documentation for 03MegaBlog_appWrite

Root Directory

  • .env - Stores secret environment variables like Appwrite API keys and endpoints. This file is not committed to version control.
  • .env.sample - An example file showing what variables are needed in .env.
  • .gitignore - Specifies files and folders that Git should ignore (e.g., node_modules, .env).
  • index.html - The main HTML entry point for the application, where the React app is mounted.
  • package.json & package-lock.json - Manages project dependencies, metadata, and scripts (e.g., dev, build).
  • vite.config.js - Configuration file for the Vite build tool and development server.
  • /public - Contains static assets that are directly served, like favicons or images.
  • /src - The main folder containing all the application's source code.

/src Directory

This is where the core logic of the React application resides.

  • main.jsx - The entry point of the React application. It renders the root App component and wraps it with the Redux Provider and React Router.
  • App.jsx - The main application component. It sets up the main layout (Header and Footer) and handles loading initial user data.
  • index.css & App.css - Global and application-level stylesheets.

  • /appwrite

    • auth.js - A service module for handling all Appwrite authentication logic (login, logout, signup, getting current user).
    • config.js - A service module for handling all Appwrite database and storage logic (create/update/delete posts, file upload/preview/delete).

  • /Components

    • This directory holds all the reusable UI components of the application.
    • index.js - Exports all components from this directory for cleaner, single-line imports elsewhere in the app.
    • RTE.jsx - A Rich Text Editor component, which wraps the TinyMCE editor and integrates it with react-hook-form.
    • /Header, /Footer - Components for the main site layout.
    • /Post-Form - Contains the PostForm.jsx component used for creating and editing blog posts.
    • Other files like Button.jsx, Input.jsx, Logo.jsx, etc., are basic, reusable UI elements.

  • /conf

    • conf.js - Exports a configuration object containing all environment variables (imported from .env). This provides a single, consistent source for configuration values like Appwrite URLs and project IDs.

  • /Pages

    • Contains components that represent full pages or views of the application (e.g., Home, Login, Signup, All Posts). These components often compose smaller UI components from the /Components directory to build the page.

  • /Store

    • This directory contains all the setup for Redux Toolkit, used for global state management.
    • authSlice.js - A Redux "slice" that defines the state and reducers related to authentication (e.g., storing user data and login status).
    • Store.js - The main Redux store configuration file, where all slices are combined.
